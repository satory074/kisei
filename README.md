# kisei（帰省 — 経路くらべ）

多モーダル（飛行機・新幹線・鉄道・フェリー・バス・車・レンタカー）の経路を実時刻表ベースで列挙し、所要時間と料金を一覧比較する静的 Web アプリ。

**公開 URL**: https://satory074.github.io/kisei/

大阪⇔大間町（下北半島）の帰省のたびに「どの組み合わせが最安/最速か」を手で列挙し直すのが面倒、という課題を解決するために作った。エンジンは汎用（データにある任意の2地点で検索可能）で、初期データは大阪⇔大間コリドーに特化している。

## 使い方

出発地・到着地・出発日時を選んで検索すると、乗り継ぎ可能な経路が一覧で出る。

- **最安 / 最速 / 出発順** でソート。**最速・最安バッジ**付き
- 左端が緑のカードは**パレート最適**（より早くかつ安い経路が存在しない）
- 各カードはレグ別のタイムラインで、**乗換・待ち時間**（フェリー待ち等）を明示
- 検索条件は URL クエリに同期されるので、**URL をそのまま共有可能**（例: `?from=osaka&to=oma&time=09:00&sort=cheapest`）

## コマンド

```bash
npm install
npm run dev        # http://localhost:4321/kisei/
npm run build      # 本番ビルド（dist/）
npm run typecheck  # astro check（scripts/ も型検査される）
npm run test       # エンジン+データ検証（smoketest）と DOM テスト（tsx + jsdom）
```

## アーキテクチャ

```
src/
├── data/network.json   # 全データ（ノード・エッジ・乗換設定・メタ）。レンダリングの単一の真実
├── engine/             # 純TS・DOM無し・Dateオブジェクト無し
│   ├── types.ts        # データ/結果の型定義
│   ├── time.ts         # 「出発日0時からの経過分」での時刻演算（"25:30"式の24時超表記対応）
│   ├── validate.ts     # network.json の構造検証（手書き。zod不使用）
│   ├── compile.ts      # 検証→正規化→隣接インデックス構築
│   ├── expand.ts       # timetable/frequency/anytime の「次に乗れる便」計算（日跨ぎはここに集約）
│   ├── search.ts       # DFS + 枝刈り（(ノード×モード構成)別の支配フロンティア）
│   ├── pareto.ts       # (到着時刻, typical運賃) の2目的パレート判定
│   └── format.ts       # 表示整形の純関数
└── app/                # 唯一のDOM層
    ├── render.ts       # 描画 + data-action イベント委譲
    ├── main.ts         # エンジンとUIの配線、URL同期
    └── url.ts          # クエリ文字列の encode/decode（純関数）
```

- **探索**: 経路はハードコードせず、エッジ（区間）のグラフを DFS で列挙。エッジごとに「乗れる最初の便」だけを分岐させる（運賃がエッジ単位なので最早便が後続便を支配する）。結果は **パレート集合 ∪ モード構成ごとのベスト**（純パレートだと2〜3件に潰れて「車ならいくら？」の比較ができないため）
- **時刻**: 全便時刻表ベース。のぞみ等の高頻度路線は `frequency`（間隔運転）、車は `anytime` で表現。乗換は「ノード種別ごとの乗換時間 + モード別の乗り込みリード（航空45分・フェリー30分など）」を加算
- **運賃**: 変動運賃（航空券）は `{low, typical, high}` の幅で持ち、一覧は typical でソートしつつ幅を表示。偽精度を出さない

## データの更新方法

`src/data/network.json` を編集して `npm test`。

- 全エッジに `source`（出典URL）と `lastUpdated`（確認日）が**必須**（validate が強制）
- 概算値・逆算値は `notes` に明記する
- フェリーは季節ダイヤあり（通常ダイヤを収載）。ダイヤ改正期（3月）に見直すこと

## 既知の制限（v1）

- **曜日・季節ダイヤ非対応**（毎日同一ダイヤとして扱う。土日減便・夏期増便は notes 参照）
- **車のフェリー航送ルート非対応**（大阪→青森FT→函館に車ごと乗る経路。車両運賃クラスが複雑なため）
- **1人旅前提**（レンタカー・車の費用は1台あたりのまま合算。`costBasis` はデータに記録済みなので人数対応は将来可能）
- 一部エッジは概算（下北交通バスの運賃、函館のタクシー料金など。notes 参照）
- 青森⇔伊丹の ANA 復路便など、時刻未確認の便は未収載
